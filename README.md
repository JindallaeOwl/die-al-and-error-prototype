# GAMZAISSAC

Phaser 3, TypeScript, Vite 기반의 웹 브라우저용 2D 탑다운 로그라이크 액션 게임 프로토타입입니다. 빠른 방 단위 전투, 회피와 4방향 사격, 랜덤 보상, 패시브 아이템 성장을 중심으로 개발하고 있습니다. 직접 제작한 도트와 라이선스를 확인한 외부 에셋을 함께 사용하며, 출처는 `ASSET_CREDITS.md`에서 관리합니다.

현재 구현 상태, 보류된 작업과 다음 개발 후보는 [`DEVELOPMENT_STATUS.md`](DEVELOPMENT_STATUS.md)에서 관리합니다.

게임 플레이는 480×272 논리 해상도와 16×16 기본 격자를 사용합니다. 화면에는 가장 가까운 픽셀 방식으로 확대하며, 창 비율이 달라도 게임 전체가 보이도록 자동으로 맞춥니다. 게임과 창의 비율이 다르면 위아래 또는 좌우에 여백이 생길 수 있습니다. 렌더 품질은 높음(4배)이 기본값이며, 설정에서 낮음(1배)·균형(2배)·높음(4배)을 바꾸면 즉시 적용됩니다.

## 픽셀 아트 제작 기준

| 용도               |        제작 크기 |
| ------------------ | ---------------: |
| 기준 해상도        |          480×272 |
| 기본 타일          |            16×16 |
| 열쇠·폭탄·코인 HUD |            16×16 |
| 수집 아이템        |            32×32 |
| 플레이어·일반 적   |         약 32×32 |
| 큰 적              | 48×48 또는 64×64 |
| 보스               |      64×64~96×96 |

에셋은 가장 가까운 픽셀 방식으로 확대하고 안티앨리어싱을 사용하지 않습니다. 중요한 HUD와 상호작용 요소를 포함한 전체 게임 영역은 창 안에 유지됩니다. 외부 에셋 출처와 라이선스는 `ASSET_CREDITS.md`에서 관리합니다.

## 실행 방법

```bash
npm install
npm run dev
```

기본 개발 서버 주소는 `http://127.0.0.1:5173`입니다.

## 개발 검사

기능을 수정한 뒤에는 전체 검사를 실행합니다.

```bash
npm run check
```

`check`는 ESLint, Vitest 단위 테스트, Prettier 형식 검사, TypeScript 및 Vite 프로덕션
빌드를 순서대로 실행합니다. 테스트를 작성하면서 반복 실행할 때는 `npm run test:watch`를
사용할 수 있습니다.

## 게임 조작

- 이동: `WASD`
- 씨앗 발사: 방향키
- 차징 빔: 보물방에서 `프리즘 창` 획득 후 방향키를 누르고 있다가 떼기
- 폭탄 설치: `E` (보유 시 현재 위치에 설치, 3초 후 폭발)
- 상점 상품 구매: 상품 가까이에서 `F`
- 미니맵 확대: `Tab`을 누르는 동안 확대
- 미니맵 확대 고정/해제: `Tab`을 짧게 누르기
- 일시정지/설정: `Esc`
- 디버그 표시: `F3`
- 게임 중 언어 전환: `L`
- 게임오버 후 타이틀로 돌아가기: `Enter` 또는 `Space`

## 현재 구현 범위

### 플레이어와 전투

- WASD 이동, 방향키 4방향 씨앗 발사와 차징 빔
- 기본 최대 체력 하트 3칸, 피격 시 반 칸 피해와 짧은 피격 무적
- 시작 자원: 열쇠 1개, 폭탄 1개, 코인 0개
- 체력, 이동속도, 공격력/공격력 배율, 사거리, 공격속도/공격속도 배율, 행운, 씨앗 속도 능력치
- 추적형, 원거리형, 돌진형 일반 적과 2층부터 등장해 처치 시 새끼 둘로 갈라지는 분열형 적
- 현재 방의 적 점수 합산과 게임오버 결과 표시
- 폭탄 설치(`E`): 3초 후 범위 피해, 적 탄환 제거와 파괴 가능 장애물 파괴

### 아이템과 보상

- 패시브 아이템 25종과 일반·고급·희귀·전설 희귀도
- 공격·방어·기동·자원 분류와 아이템별 고유/중첩 획득 제한
- 전투·상점·보물·보스방 전용 드랍 테이블과 행운 기반 전투방 아이템 드랍
- 두 아이템 조합으로 한 번만 발동하는 시너지 6종
- 쿼드샷(4발 부채꼴), 메가씨드(초과 피해 관통), 이쑤시개(공속·씨앗 속도 및 붉은 씨앗)
- 쿼드샷의 추가 눈과 이쑤시개가 플레이어 외형에 표시
- `프리즘 창`과 쿼드샷을 함께 보유하면 차징 빔이 부채꼴 4방향으로 발사
- 공격력과 공격속도 계열 능력치가 빔 피해와 차징 시간에 적용
- 아이템 획득 시 이름과 설명이 스크롤 형태의 알림으로 표시
- 전투방 클리어 보상: 하트, 열쇠, 폭탄, 1코인, 5코인 또는 상자
- 열쇠, 폭탄과 코인은 각각 최대 99개 보유
- 체력이 가득 찬 상태의 하트와 보상 상자를 몸으로 짧게 밀 수 있음
- 열린 상자는 사라지지 않고 열린 모습과 위치를 유지
- 줍지 않은 방 보상과 아이템은 방을 나갔다 돌아와도 종류·상태·위치를 유지

### 던전과 방

- 시작방, 전투방, 상점방, 보물방과 보스방
- 1층 기준 시작방 1개, 전투방 3개, 상점방 1개, 보물방 1개, 보스방 1개
- 시작방은 최소 2개의 실제 이동 가능한 출구 보장
- 층이 오를수록 전투방 수가 늘고 적 체력과 점수가 층마다 16% 증가
- 1층에서는 상점방과 보물방이 열려 있고, 2층부터 입장 시 열쇠 1개 사용
- 방 입장 후 플레이어 0.6초 보호, 적 AI 0.4초 대기와 출입구 안전거리 스폰
- 전투방 클리어 후 0.5초가 지나면 문이 열려 문 앞에서 즉시 이동되는 상황 방지
- 이동과 씨앗을 막는 파괴 가능 나무 상자: 씨앗·차징 빔으로 파괴하거나 폭탄으로 즉시 파괴
- 나무 상자 파괴 시 낮은 확률로 1코인 또는 5코인 드랍
- 상점방 NPC, 네 개의 랜덤 상품과 상품 한 개의 확률적 50% 할인
- 상점 입장 시 상인 말풍선 표시, 접촉 시 최대 3픽셀 밀렸다 원위치로 복귀
- 보물방 중앙에 전용 드랍 테이블의 랜덤 아이템 생성

### 보스와 층 진행

- placeholder 보스 `Fault Warden`, `ROOT KERNEL`
- 플레이어 방향 탄환 버스트, 짧은 돌진과 체력 50% 이하 2페이즈 전환
- `ROOT KERNEL`의 체크섬 십자탄, 패킷 장막과 일부가 끊어진 원형탄
- 보스 체력바와 2페이즈 상태 표시
- 보스 처치 후 공격력·공격속도·체력 등 순수 능력치 전용 보상
- 보스 보상 `빨간 버섯`(`red-mushroom`): 최대 하트 1칸 증가 및 하트 1칸 회복
- 보스 처치 시 다음 층 구멍 생성, 직접 들어가면 다음 층으로 이동
- 홀수 층은 `Fault Warden`, 짝수 층은 `ROOT KERNEL` 배치

### UI와 오디오

- 타이틀, 설정, 일시정지, 게임오버와 타이틀 복귀 화면
- 한국어/영어 전환과 저장, 런타임 언어 변경
- 하트·자원·능력치 HUD, 기본/확장 미니맵과 디버그 오버레이
- 확장 미니맵에 플레이 시간과 점수 표시
- 타이틀·일반방·상점·보스방 전용 루프 배경음악과 방 종류별 크로스페이드
- 일시정지와 브라우저 창 이탈 중에도 배경음악 재생 유지
- WebAudio oscillator 기반 procedural 효과음

### 개발자 도구

- 비밀번호 인증 방식의 개발자 콘솔
- 명령어 자동완성, 이전 명령어 탐색, 마우스 이동과 창 크기 조절
- 인증 후 `F2`로 이미지·아이템 번호 기반 아이템 선택기 열기/닫기
- 관리자 명령을 사용한 런은 `ADMIN`으로 기록하여 향후 랭킹 등록 대상에서 제외
- 비밀번호와 비밀 입력 순서는 공개 문서에 기록하지 않음

## 버그 수정 메모

- 적 탄환/플레이어 씨앗 충돌 시 `consume()` 단계로 body를 먼저 비활성화해 중복 overlap 처리를 막았습니다.
- 플레이어 피격, 적 피격, 탄환 벽 충돌 콜백에 active/body guard를 추가했습니다.
- WebAudio 호출 실패가 게임 루프를 끊지 않도록 `AudioSystem`에 안전 처리를 추가했습니다.
- 히트스톱은 물리 월드 pause/timeScale을 건드리지 않고 짧은 카메라 flash만 사용합니다.
- 적 피격 플래시는 `setTintFill` 대신 짧고 약한 tint로 낮췄습니다.

## 방 구조

1스테이지는 빠르게 테스트할 수 있도록 압축되어 있습니다.

- 시작방 1개
- 전투방 3개
- 보물방 1개
- 상점방 1개
- 보스방 1개

시작방은 최소 2개의 실제 이동 가능한 출구를 갖습니다. 2층부터는 `DungeonManager`가 층
번호에 따라 전투방 수를 늘리며, 현재 준비된 두 보스가 층마다 번갈아 등장합니다.

## 보스 보상 추가 방법

1. `src/data/items.ts`의 `PASSIVE_ITEMS`에 보상 아이템을 추가합니다.
2. 아이템의 `dropSources`에 `'boss'`를 포함합니다.
3. `src/data/bossRewards.ts`의 `BOSS_REWARD_ITEM_IDS`에 아이템 ID를 추가합니다.

목록에 아이템이 하나 이상 있으면 보스 처치 후 다음 층 구멍 아래쪽에 보상이 생성됩니다.
보상을 줍지 않고 방을 나갔다 돌아오면 같은 아이템이 다시 나타나며, 획득한 뒤에는 중복 생성되지 않습니다.
공격 형태를 바꾸는 패시브와 능력 해금 아이템은 목록에 실수로 넣어도 보스 보상에서 자동 제외됩니다.

## 개발 Phase

### Phase 0 — 개발 기준선 (완료)

- README와 실제 구현 동기화
- ESLint, Vitest, Prettier, 프로덕션 빌드를 묶은 자동 검사 구성

### Phase 1 — 플레이 루프 구조 정리 (완료)

- 아이템 획득 규칙을 `GameScene`에서 `ItemSystem`으로 분리
- 아이템 능력치 적용과 능력 중복 해금 방지 테스트 추가
- 보상 적용 규칙을 `GameScene`에서 `RewardSystem`으로 분리
- 보상 최대 보유량, 상자 회복과 인벤토리 반영 테스트 추가
- 보물방 열쇠 소비와 방 이동 판단을 `RoomNavigationSystem`으로 분리
- 층 증가와 체력 회복을 `RunProgressionSystem`으로 분리
- 플레이어·적 탄환, 접촉 피해, 빔과 장애물 충돌을 `CombatCollisionSystem`으로 분리
- 적 탄환의 수동 충돌 반경 경계 테스트 추가
- 폭탄 설치·쿨다운·폭발 피해·탄환 및 장애물 제거를 `BombSystem`으로 분리
- 폭탄 소비, 쿨다운과 폭발 반경 경계 테스트 추가
- 방 이동·층 이동의 임시 오브젝트 정리와 방 상태 복원을 `RoomTransitionSystem`으로 분리
- 방 종류별 전환 시간과 안내 문구 규칙 테스트 추가
- 타이틀·일시정지 화면의 설정 항목 생성과 변경 규칙을 공통 `SettingsMenu`로 분리
- 사운드, 음량, 화면 흔들림과 렌더 품질 순환 규칙 테스트 추가
- 보스 체력바 생성·표시·갱신을 전용 `BossHud`로 분리
