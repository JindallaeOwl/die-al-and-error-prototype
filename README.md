# GAMZAISSAC

Phaser 3, TypeScript, Vite 기반의 웹 브라우저용 2D 탑다운 로그라이크 액션 게임 프로토타입입니다. 빠른 방 단위 전투, 회피와 4방향 사격, 랜덤 보상, 패시브 아이템 성장을 중심으로 만들었고, 캐릭터/적/아이템/명칭/연출은 모두 오리지널 placeholder로 구성했습니다.

게임 플레이 좌표계는 960×640을 유지하면서 고해상도 내부 캔버스로 렌더링한 뒤 화면 크기에 맞춰 표시합니다. 설정의 렌더 품질(낮음/균형/높음)은 다음 실행부터 적용됩니다.

## 배포

- 주 배포처: [Vercel](https://ryuseungyeop09.vercel.app/) (GitHub main 브랜치 연동)
- 보조 배포처: Netlify

main에 푸시하면 Vercel 운영 배포가 자동으로 갱신됩니다.

## 실행 방법

```bash
npm install
npm run dev
```

기본 개발 서버 주소는 `http://127.0.0.1:5173`입니다.

## 개발 검사

기능을 수정한 뒤에는 전체 검사를 실행합니다.

```bash
npm run check
```

`check`는 ESLint, Vitest 단위 테스트, Prettier 형식 검사, TypeScript 및 Vite 프로덕션
빌드를 순서대로 실행합니다. 테스트를 작성하면서 반복 실행할 때는 `npm run test:watch`를
사용할 수 있습니다.

## 메인메뉴

- 시작: 게임을 시작합니다.
- 설정: 언어, 사운드, 효과음 음량, 화면 흔들림, 렌더 품질, 전체 화면을 바꿉니다.
- 나가기: 브라우저 탭을 닫으라는 안내를 표시합니다.

메뉴 조작:

- 선택 이동: `위/아래 방향키` 또는 `W/S`
- 선택 실행: `Enter` 또는 `Space`
- 마우스 hover/click 지원

## 게임 조작

- 이동: `WASD`
- 씨앗 발사: 방향키
- 차징 빔: 보물방에서 프리즘 빔 획득 후 방향키를 누르고 있다가 떼기
- 폭탄 설치: `E` (보유 시 현재 위치에 설치, 3초 후 폭발)
- 일시정지/설정: `Esc`
- 디버그 표시: `F3`
- 게임 중 언어 전환: `L`
- 게임오버 후 재시작: `Enter` 또는 `Space`

## 현재 구현 범위

- 타이틀 메뉴, 설정 메뉴, 게임 시작, 게임오버, 재시작
- 한국어/영어 i18n 구조와 런타임 언어 전환
- WASD 이동과 방향키 4방향 발사
- 플레이어 체력, 이동속도, 공격력/공격력 배율, 사거리, 공격속도/공격속도 배율, 행운, 씨앗 속도
- 기본 공격 콘셉트: 감자가 눈에서 씨앗을 발사
- 신규 패시브: 쿼드샷(4발 부채꼴), 메가씨드(초과 피해 관통), 이쑤시개(공속·씨앗 속도 및 붉은 씨앗)
- 쿼드샷의 추가 눈과 이쑤시개가 플레이어 외형에 표시
- 프리즘 빔 사용 중에는 씨앗 개수·퍼짐·크기·색·초과 피해 관통·씨앗 속도 효과가 비활성화되며, 공격력과 공격속도 계열은 빔 피해 및 차징 시간에 적용
- 피격 무적 시간과 짧은 시각 피드백
- 추적형, 원거리형, 돌진형 적
- 보스방과 placeholder 보스 `Fault Warden`, `ROOT KERNEL`
- 보스 패턴: 플레이어 방향 탄환 버스트, 짧은 돌진, 50% 이하 2페이즈 전환
- `ROOT KERNEL` 패턴: 체크섬 십자탄, 패킷 장막, 일부가 끊어진 원형탄, 50% 이하 2페이즈 전환
- 보스 체력바, 2페이즈 상태 표시, 스테이지 클리어 메시지
- 보스 처치 시 보스방 바닥에 다음 층 구멍 생성: 다른 방 클리어 여부와 관계없이 직접 들어가면 다음 층으로 이동
- 층이 오를수록 전투방 수가 늘고, 적 체력과 점수가 층마다 16% 증가
- 홀수 층은 `Fault Warden`, 짝수 층은 `ROOT KERNEL` 보스방 배치
- 시작방 2개 출구
- 1스테이지 기준 짧아진 방 구성
- 시작방, 전투방, 보상방, 열쇠로 여는 보물방, 보스방
- 전투 방 클리어 보상: 열쇠/폭탄/코인/상자
- 열쇠, 폭탄, 코인 최대 99개 보유
- 폭탄 설치(`E`): 현재 위치에 설치 후 3초 뒤 폭발하며, 범위 안의 적에게 광역 데미지를 주고 적 탄환과 파괴 가능 장애물을 제거
- 전투방 파괴 가능 장애물(나무 상자): 이동/씨앗을 막는 엄폐물. 플레이어 씨앗 여러 발로 격파하거나 폭탄으로 즉시 파괴 가능
- 보물방 중앙 랜덤 스탯 아이템
- 보물방 랜덤 보상에 차징 빔 아이템 `프리즘 빔` 포함
- HUD, 미니맵, 디버그 오버레이
- WebAudio oscillator 기반 procedural 효과음

## 버그 수정 메모

- 적 탄환/플레이어 씨앗 충돌 시 `consume()` 단계로 body를 먼저 비활성화해 중복 overlap 처리를 막았습니다.
- 플레이어 피격, 적 피격, 탄환 벽 충돌 콜백에 active/body guard를 추가했습니다.
- WebAudio 호출 실패가 게임 루프를 끊지 않도록 `AudioSystem`에 안전 처리를 추가했습니다.
- 히트스톱은 물리 월드 pause/timeScale을 건드리지 않고 짧은 카메라 flash만 사용합니다.
- 적 피격 플래시는 `setTintFill` 대신 짧고 약한 tint로 낮췄습니다.

## 방 구조

1스테이지는 빠르게 테스트할 수 있도록 압축되어 있습니다.

- 시작방 1개
- 전투방 3개
- 보물방 1개
- 보상방 1개
- 보스방 1개

시작방은 최소 2개의 실제 이동 가능한 출구를 갖습니다. 2층부터는 `DungeonManager`가 층
번호에 따라 전투방 수를 늘리며, 현재 준비된 두 보스가 층마다 번갈아 등장합니다.

## 폰트

CSS와 Phaser Text는 시스템 폰트 fallback을 사용합니다.

```text
Galmuri11, DungGeunMo, NeoDunggeunmo Pro, DOSGothic, DNFBitBitv2,
Pretendard, Noto Sans KR, Malgun Gothic, Inter, Segoe UI, Arial, sans-serif
```

외부 폰트를 다운로드하지 않으므로, 사용자 PC에 픽셀 폰트가 설치되어 있지 않으면 일반 한글 fallback 폰트로 보입니다. 나중에 고정 픽셀 폰트가 필요하면 `assets/fonts`에 웹폰트를 포함하고 공통 fontFamily만 교체하면 됩니다.

## 주요 구조

```text
src/
  config/   밸런스 값, 피드백 값, 화면 크기, 텍스처 키

  entities/ 플레이어, 적, 씨앗/적 탄환, 빔, 문, 아이템/보상 픽업
  i18n/     한국어/영어 번역과 fallback 처리
  scenes/   Boot, Title, Game, GameOver
  systems/  던전, 방 제어, 아이템, 보상, 인벤토리, 이펙트, 오디오
  ui/       HUD와 미니맵
  utils/    방향, 수학, 랜덤 유틸
```

## 개발 Phase

### Phase 0 — 개발 기준선

- README와 실제 구현 동기화
- ESLint, Vitest, Prettier, 프로덕션 빌드를 묶은 자동 검사 구성
- 인벤토리, 아이템, 보상, 던전 생성 핵심 로직 단위 테스트
- 던전·아이템·보상 RNG를 주입 가능하게 만들어 랜덤 결과를 테스트에서 재현

### Phase 1 — 플레이 루프 구조 정리 (완료)

- 아이템 획득 규칙을 `GameScene`에서 `ItemSystem`으로 분리
- 아이템 능력치 적용과 능력 중복 해금 방지 테스트 추가
- 보상 적용 규칙을 `GameScene`에서 `RewardSystem`으로 분리
- 보상 최대 보유량, 상자 회복과 인벤토리 반영 테스트 추가
- 보물방 열쇠 소비와 방 이동 판단을 `RoomNavigationSystem`으로 분리
- 층 증가와 체력 회복을 `RunProgressionSystem`으로 분리
- 플레이어·적 탄환, 접촉 피해, 빔과 장애물 충돌을 `CombatCollisionSystem`으로 분리
- 적 탄환의 수동 충돌 반경 경계 테스트 추가
- 폭탄 설치·쿨다운·폭발 피해·탄환 및 장애물 제거를 `BombSystem`으로 분리
- 폭탄 소비, 쿨다운과 폭발 반경 경계 테스트 추가
- 방 이동·층 이동의 임시 오브젝트 정리와 방 상태 복원을 `RoomTransitionSystem`으로 분리
- 방 종류별 전환 시간과 안내 문구 규칙 테스트 추가
- 타이틀·일시정지 화면의 설정 항목 생성과 변경 규칙을 공통 `SettingsMenu`로 분리
- 사운드, 음량, 화면 흔들림과 렌더 품질 순환 규칙 테스트 추가
- 보스 체력바 생성·표시·갱신을 전용 `BossHud`로 분리

### 이후 Phase

1. 아이템 희귀도, 드랍 테이블, 시너지와 빌드 다양화
2. 층별 적·방·보스 보상 콘텐츠 확장
3. 저장/로드, 설정, 키 리바인딩과 접근성 개선
4. 실제 스프라이트·사운드 교체, 성능 점검과 배포 QA

## 저작권/유사성 방지 메모

이 프로젝트는 장르적 구조만 참고하며, 특정 상용 게임의 캐릭터, 적, 아이템, 방 테마, UI 표현, 스토리, 명칭, 아트, 사운드, 연출을 직접 복제하지 않습니다.
