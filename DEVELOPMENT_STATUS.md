# GAMZAISSAC 개발 현황

이 문서는 다른 컴퓨터나 새로운 AI 대화에서 작업을 이어갈 때 현재 상태를 빠르게 파악하기 위한 개발용 인수인계 문서입니다. 일반적인 실행 방법과 플레이 조작은 [`README.md`](README.md), 외부 에셋 출처는 [`ASSET_CREDITS.md`](ASSET_CREDITS.md)를 우선 확인합니다.

마지막 코드 대조일: 2026-07-23

## 현재 개발 단계

- Phase 0 개발 기준선: 완료
- Phase 1 플레이 루프 구조 정리: 완료
- Phase 1.5 픽셀 아트 기술 기준선: 완료
- 아이템 희귀도·드랍 테이블·시너지 확장: 1차 완료
- 현재 위치: 층별 적·방·보스 보상 콘텐츠 확장 준비 단계

## 최근 완료된 작업

- 2층부터 등장하는 분열형 적 추가: 부모(`splitter`) 처치 시 새끼(`splitterling`) 둘로 갈라지며, 새끼는 다시 분열하지 않음
- 증원·분열 계산을 `EnemyReinforcementRules`로 분리하고 2층 등장 조건과 자식 생성 단위 테스트 추가
- 개발자 콘솔 `kill` 명령이 분열로 생성된 새끼까지 모두 처치하도록 수정
- 패시브 아이템을 25종으로 확장하고 아이템 번호, 희귀도, 분류, 최대 중첩과 방별 드랍 출처 추가
- 두 아이템 조합으로 한 번만 발동하는 시너지 6종 추가
- 전투방 아이템 드랍과 방 종류별 아이템 풀 개선
- 방 클리어 보상, 부순 장애물 보상과 미획득 아이템의 방 재입장 상태 유지
- 체력이 가득 찬 하트와 보상 상자 밀기, 열린 상자 상태 유지
- 아이템 획득 스크롤 알림과 연속 획득 대기열 추가
- 개발자 콘솔 로직을 `DeveloperConsoleController`로 분리
- 이미지·아이템 번호 기반 개발자 아이템 선택기 추가
- 개발자 콘솔과 아이템 선택기 창의 드래그·크기 조절·위치 저장 추가
- 인증 후 `F2`로 아이템 선택기 열기/닫기 추가

## 구현된 게임 흐름

1. 타이틀에서 게임을 시작합니다.
2. 시작방에서 전투방, 상점방과 보물방을 탐색합니다.
3. 전투방을 클리어하면 자원 보상 또는 아이템이 확률적으로 생성됩니다.
4. 보스방에서 층별 보스를 처치하면 순수 능력치형 보상과 다음 층 구멍이 생성됩니다.
5. 구멍에 들어가면 층이 증가하고 던전이 새로 생성됩니다.
6. 체력이 모두 소진되면 게임오버 결과를 표시한 뒤 타이틀로 돌아갑니다.

## 상태 유지 범위

### 현재 런 안에서 유지됨

- 현재 층, 점수, 클리어 방 수
- 체력과 능력치
- 열쇠, 폭탄과 코인
- 획득한 아이템, 해금 능력과 발동한 시너지
- 방별 클리어 여부와 특수방 잠금 해제 여부
- 방에 남겨둔 아이템과 보상의 종류·상태·위치
- 상점 상품, 구매 여부와 할인 상태
- 보스 보상 선택 및 획득 여부

### 브라우저를 닫거나 새로고침해도 유지됨

- 언어
- 사운드, 효과음 음량, 화면 흔들림과 렌더 품질 설정
- 개발자 콘솔과 아이템 선택기 창 위치·크기

### 아직 유지되지 않음

- 진행 중인 런 전체 저장/불러오기
- 최고 점수와 플레이 기록
- 게임 포럼 라운지 랭킹 등록

`runState.adminUsed`와 랭킹 등록 가능 여부 판단 함수만 준비되어 있습니다. 실제 외부 랭킹 API 연동은 아직 구현되지 않았습니다.

## 주요 안전 처리

- 방 입장 시 플레이어 보호 600ms
- 방 입장 시 적 AI 대기 400ms
- 적을 출입구와 장애물에서 떨어진 안전 위치에 배치
- 방 클리어 후 문 개방 500ms 지연
- 탄환 충돌 전 active/body 검사와 중복 소비 방지
- 게임오버·일시정지 나가기의 타이틀 전환에 브라우저 타이머 fallback 적용
- Phaser Scene 재사용 시 런 전용 상태를 명시적으로 초기화
- 오디오 호출 실패가 게임 루프를 중단하지 않도록 예외 처리

## 개발자 도구

- 게임 중 개발자 콘솔을 열 수 있습니다.
- 콘솔은 비밀번호 인증 후 명령을 실행합니다.
- 자동완성, 명령 기록, 마우스 이동과 크기 조절을 지원합니다.
- 인증 후 `F2`로 아이템 선택기를 열 수 있습니다.
- 아이템 선택기는 아이콘, `ID: 001` 형식의 번호와 이름을 표시합니다.
- 실제 게임 상태를 바꾸는 관리자 명령을 사용하면 화면에 `ADMIN`이 표시되고 해당 런은 랭킹 등록 불가 상태가 됩니다.

비밀번호 원문, 해시 생성 과정과 비밀 입력 순서는 이 문서에 기록하지 않습니다. 필요한 경우 현재 구현 코드를 직접 확인하되 공개 문서나 커밋 메시지에 비밀번호 원문을 남기지 않습니다.

## 새 작업을 시작하기 전 확인 절차

1. `git status --short --branch`로 커밋되지 않은 변경을 확인합니다.
2. `README.md`와 이 문서를 읽습니다.
3. `git log --oneline -10`으로 문서 작성 이후의 최근 커밋을 확인합니다.
4. 요청한 기능과 관련된 실제 코드를 읽고 문서가 여전히 맞는지 검증합니다.
5. 작업 후 `npm run check`를 실행합니다.
6. 직접 플레이가 필요한 기능은 자동 검사와 별도로 브라우저에서 확인합니다.

문서보다 실제 코드와 Git 기록이 우선입니다. 기능을 추가하거나 동작을 바꾼 경우 같은 작업에서 README 또는 이 문서도 함께 갱신합니다.
